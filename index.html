Vue学习总结

发表于 2018-03-08

Vue是我最开始接触到的前端框架，相比Angula和React更容易掌握。以下是我对Vue的阶段性总结以及在使用Vue过程中我遇到的一些问题：
Vue是负责将view（静态界面）及Model（数据源）结合起来，实例化一个Vue配置参数el指向html元素，data参数指向数据

<template>
    <div id="app">{{data}}<div>
</template>
<script type="text/javascirpt">
    new Vue({
        el : '#app',
        data : data
    })
</script>

实例

V-model：

Vue的核心就是数据绑定即V-model，我们可以将html与数据绑定起来，只要数据发生改变，html立即更新；由from表单等带来的数据改变，数据相应字段也会发生相应改变。

##实例生命周期钩子函数：
每个 Vue 实例在被创建之前，都要经过一系列的初始化过程 - 例如，Vue实例需要设置数据观察、编译模板、在DOM 挂载实例，以及在数据变化时更新DOM。在这个过程中，Vue 实例还会调用执行一些生命周期钩子函数，这样用户能够在特定阶段添加自己的代码：

beforeCreate: function () {
    console.group('beforeCreate 创建前状态===》');
},
created: function () {
    console.group('created 创建完毕状态===》');
},
beforeMount: function () {
    console.group('beforeMount 挂载前状态===》');
},
mounted: function () {
    console.group('mounted 挂在结束状态===》');
},
beforeUpdate: function () {
    console.group('beforeUpdate 更新前状态===》');
},
Updated: function () {
    console.group('Updated 更新完成状态===》');
},
beforeDestroy: function () {
    console.group('beforeDestroy 销毁前状态===》');
},
destroyed: function () {
    console.group('destroyed 销毁完成状态===》');
},

实例

Watch：

先在computed中创建一个方法后可在watch中监听该数据的变化进行不同的操作。

computed:{
    path: function () {
        return this.$router.path;
    }
}，
watch: {
    path: function () {
        if (this.$route.path === '/'){
            this.tiggle();
        } else {
            this.normal();
        }
    }
}

实例

组件Components：

组件是Vue编译器将行为附加到自定义元素，所有组件也是Vue实例，因此接受相同的选项对象并提供相同的生命周期钩子函数。
在使用组件前需要先在index.js中注册全局组件，

import map from './components/map/map';
const compoents = {
    map
};

实例

在组件中导出：

const prefixCls = 'v-map';
export defauil {
    name: prefixCls
}

实例

在其他子组件页面中直接引入即可随意使用：

<div>
    <v-map></v-map>
<div>

实例

v-for是Vue中做循环的，值可以使数组、对象、数值三种类型

<swiper-slide v-for="item in teachers" :key="item.id">
    <a>
        <div class="swipe-img"><img :src="item.img" alt=""></div>

</swiper-slide >

实例

Props属性：

子组件可通过props接收父组件所传递的数据，一般情况下不要在子组件中改变父组件传递过来的props。
Sync父子组件的双向绑定:
.sync修饰符所提供的功能。当一个子组件改变了一个 prop 的值时，这个变化也会同步到父组件中所绑定，就是说我们可以直接在我们需要传的prop后面加上.sync。
父组件：

<v-loading :appear.sync=""></v-loading>

实例

会自动拓展为一个额外的v-on侦听器：

<v-loading :appear="loginshow" @ update:appear="val =>loginshow=val" ></v-loading>

子组件：

props: {
    appear: {
        defalut: false
    }
}

实例
